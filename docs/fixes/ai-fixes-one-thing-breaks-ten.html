AI Fixes One Thing and Breaks Ten Others

By Sandeep Roy · June 6, 2026 · 5 min read

You report one bug. The login button is misaligned. The AI fixes it — and on its way out, "refactors" the form component, renames a prop three other screens depend on, and changes the validation logic so empty emails now slip through.

One thing fixed. Ten things broken. You spend the rest of the afternoon undoing help.

If that feels familiar, it's because it's the single most-repeated complaint in vibe coding. As one widely-shared roundup put it bluntly: "it will fix one thing but destroy 10 other things in your code" (morphllm / Reddit). And it's not just hobbyists — that same write-up notes 16 of 18 CTOs surveyed reported an AI-caused production disaster.

Why a "Small Fix" Turns Into a Blast Radius

An AI agent doesn't have a concept of "the smallest change that solves this." When you ask it to fix one thing, it sees the whole file, the whole module, the whole repo — and it's eager. It "improves" adjacent code because improving looks helpful. It has no fence telling it this part is done, do not touch it.

You might think your rules file covers this. You wrote "don't change the public API," "don't touch the payment module," "keep the validation strict." But a rules file only suggests. The model reads it, then weighs it against your latest prompt — and "fix the login button" feels more urgent than a rule it skimmed 40 messages ago. Worse, once the file gets long, the rules at the bottom get followed "intermittently or not at all" (Hacker News).

So there's nothing actually holding the line between the one file you wanted changed and the ten you didn't. The collateral edits land in the same diff, and you only catch them if you read every line.

The Fix: Fence the Code That Must Not Change

SpecLock is an open-source AI constraint engine. It reads the rules you already wrote and enforces them as runtime hard constraints — so the agent can fix the one thing you asked for and is blocked from quietly rewriting the rest.

npx speclock protect

That one command reads your existing CLAUDE.md / .cursorrules / AGENTS.md, turns the rules into locks, and starts enforcing across Claude Code, Cursor, Windsurf, Cline, and Copilot. No config, no new files to write. It runs as an MCP server with 51 tools, including blast_radius — which shows you exactly how far a proposed change reaches before it lands.

Before and After: Containing the Damage

Say SpecLock has these locks, derived from your rules:

Lock 1: Do not modify the public API surface (src/api/**)
Lock 2: Payment logic is frozen (src/billing/**)
Lock 3: Validation must stay strict (no relaxing email/required checks)

Before SpecLock

You ask: "Fix the misaligned login button." The agent fixes the CSS — and also edits src/api/user.ts, renames a prop, and loosens the email validator. All in one 9-file diff. It merges. Three screens break in production by morning.

After SpecLock — the collateral edits are stopped: 
🔒 SpecLock reviewed this changeset

  ✓ src/components/LoginButton.css   allowed (your request)
  ✗ src/api/user.ts                  BLOCKED
        violates Lock 1: public API surface is frozen
  ✗ src/auth/validate.ts             BLOCKED
        violates Lock 3: relaxes email validation (HIGH)

  1 change applied. 2 out-of-scope changes blocked.
  The fix you asked for shipped. The damage didn't.

The button gets fixed. The two changes you never asked for — the ones that would have broken three other screens — are stopped at the gate, each tagged with the exact rule it violated. You review them deliberately instead of discovering them in an incident.

It catches the euphemisms too. "Simplify the validation," "clean up the API helper," "refactor billing into a tidier shape" — SpecLock's semantic detection reads what the change does, so a relaxed check disguised as a cleanup still trips the lock.

Keep the Speed. Lose the Whack-a-Mole.

The promise of AI coding was supposed to be faster, not fix-one-break-ten followed by an hour of cleanup. SpecLock gives you back the part that was missing: a hard line around the code that's already correct, so the agent can move fast everywhere it's actually safe to.

And every time it blocks a stray edit, SpecLock keeps a receipt. Run speclock wins for a shareable "Save Receipt" of exactly what it stopped — the disaster you didn't have.

Lock the files it keeps breaking. Let it work everywhere else.
