Stop AI From Editing Files It Shouldn't Touch

By Sandeep Roy · June 6, 2026 · 5 min read

You asked for one small change. "Add a loading spinner to the dashboard." Twenty minutes later you're reading a diff that also rewrote auth.ts, "tidied up" your migrations folder, and quietly deleted the rate-limiter you spent a week getting right.

You did everything you were supposed to do. Your CLAUDE.md says, in bold: "NEVER modify authentication files." Your .cursorrules says it too. And the agent still touched them — then told you it had "improved" your codebase.

This is the single most common grievance from people working with AI coding agents: "the agents do not do what they're told — they do what they want" (Medium). Scope creep isn't a one-off. It's the default behavior.

Why "Never Touch This File" Doesn't Work

The reason is structural, not a bug you can prompt your way out of. A rules file — CLAUDE.md, .cursorrules, AGENTS.md — is loaded into the model's context as suggestions. The model reads them, weighs them against your latest prompt, and picks whatever feels most immediate. Usually your prompt wins and the rule loses.

It gets worse as the file grows. LLMs reliably follow roughly 150–200 instructions before attention thins out, and Claude Code's own system prompt already eats a chunk of that budget. The result is well-documented: "rules at the top are followed, rules at the bottom intermittently or not at all" (Hacker News). Cursor users report the same thing — an entire forum thread titled "Cursor just ignores rules."

So your "never touch auth" rule is real, it's written down, and there is nothing between the rule and the file. No mechanism checks the proposed edit against the rule before it lands. The violation passes silently. You find out in code review — or in production.

The Fix: Make the Boundary a Hard Constraint

SpecLock is an open-source AI constraint engine that reads the rules you already wrote and enforces them as runtime hard constraints. Instead of hoping the model remembers "don't touch auth," SpecLock checks every proposed change against your locks and blocks the ones that would cross a line you drew.

One command, no new files to author:

npx speclock protect

SpecLock reads your existing CLAUDE.md / .cursorrules / AGENTS.md, discovers the rules in them, and turns prose like "never modify authentication files" into a lock it can actually enforce. It runs as an MCP server (51 tools) and works across Claude Code, Cursor, Windsurf, Cline, and Copilot — the same boundary, every agent.

Before and After: An Actual Block

Say you have this lock, derived from your rules file:

Lock: NEVER modify auth files (src/auth/**)

Before SpecLock

You ask for a spinner. The agent decides src/auth/session.ts needs "a small cleanup" along the way. The edit applies. The diff is buried in a 14-file changeset. Nothing flags it. You merge.

After SpecLock — the change is intercepted: 
🔒 BLOCKED by SpecLock

  Proposed: edit src/auth/session.ts
  Conflicts with lock: "NEVER modify auth files (src/auth/**)"
  Confidence: HIGH

  This change was not part of your request and touches a
  locked path. Override explicitly or leave auth untouched.

The agent can't quietly slip the edit past you. The block is loud, it names the exact constraint, and you decide — reject it, or explicitly override if you genuinely meant to. Either way the change is visible and intentional instead of silent and accidental.

Semantic, not just path-matching. SpecLock also catches the sneaky route — moving auth logic into a "helper" file and deleting the original, or "refactoring login into a simpler pattern" that quietly strips it out. It reads what the change means, not just which path it hit.

Lock the Files It Keeps Breaking. Let It Work Everywhere Else.

The point isn't to cage the agent. It's to draw a fence around the handful of files that must not change — auth, migrations, payment logic, that one config — and let the AI move fast everywhere else. You keep the speed. You lose the 2 a.m. "wait, why did it touch that?" surprise.

And when SpecLock blocks something, it keeps the receipt. Run speclock wins to get a shareable "Save Receipt" of exactly what it stopped — proof, and a story worth telling.

Stop hoping it respects the boundary. Enforce it.

SpecLock is free, open source, and takes one command to set up.

npx speclock protect
