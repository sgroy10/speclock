4 locks set. 4 violations attempted. 4 blocked.

A real test on a client jewelry wholesale project. SpecLock caught every constraint violation before any code was changed.

# Session 1: Developer sets project constraints
$ speclock add_lock "Colors: Gold #C5A572, Black #1A1A1A only. NO bright colors, NO neon."
$ speclock add_lock "Tagline: Crafted in Guangzhou. Trusted Worldwide. Do NOT change."
$ speclock add_lock "Contact must include WeChat QR + WhatsApp + Alibaba."
$ speclock add_lock "All prices in USD. This is B2B wholesale, NOT retail."

# Session 2: Context compacted. New AI session. Client requests changes.
# SpecLock session_briefing loads all locks automatically.

Checking: "Add blue gradient and neon green accent buttons"

Checking: "Change the tagline to something more modern"
BLOCKED [HIGH 100%] Synonym match: change/modify/alter/rewrite

Checking: "Remove WeChat from the contact section"
BLOCKED [HIGH 100%] Destructive action against locked constraint

Checking: "Convert all prices from USD to CNY"
BLOCKED [HIGH 100%] Violates: All prices in USD

Result: 4/4 violations caught. Zero damage to codebase.
The Problem

Every AI coding session starts from zero. Yours won't.

🧠

Session Amnesia

Your AI forgets everything when you start a new chat. Architecture decisions, constraints, what you tried before — all gone.

🔀

Semantic Drift

AI slowly diverges from your original requirements. By session 5, it's building something completely different than what you specified.

💥

Constraint Violations

"Don't use Redux" — you said it 10 times. The AI adds Redux anyway because it has no memory of your non-negotiable rules.

🔄

Repeat Yourself Loop

You paste the same context, the same rules, the same architecture docs at the start of every single session. Hours wasted.

👻

Multi-Agent Chaos

Using Claude + Cursor + Copilot? None of them know what the others decided. You're the only thread of continuity.

🎯

Vibe Coder Pain

Non-technical founders suffer most. They can't re-explain complex requirements. SpecLock remembers so they don't have to.

Connect

Works where you work.

Connect SpecLock to any MCP-compatible platform in under 2 minutes. Here's how for each one.

Lovable — Custom MCP Server

Easy — 2 min
1

Open Connectors Settings

In your Lovable workspace, go to Settings → Connectors → Personal connectors

2

Add New MCP Server

Click "New MCP server". Enter the following:

Server name: SpecLock Server URL: https://speclock-mcp-production.up.railway.app/mcp Auth: No authentication
3

Start Using It

In any Lovable project, click + in the prompt box → Connectors → enable SpecLock. Then tell the AI: "Initialize SpecLock and set a goal for this project"

Requires Lovable paid plan (Pro $25/mo) for custom MCP servers. Prebuilt connectors available on all plans.

Claude Code

Easy — 1 min
1

Install via Smithery

Run this in your terminal:

npx @anthropic-ai/smithery-cli install sgroy10/speclock --client claude-code
2

Or add to ~/.claude.json manually

{ "mcpServers": { "speclock": { "command": "npx", "args": ["-y", "speclock", "serve", "--project", "."] } } }

Cursor

Easy — 2 min
1

Open MCP Settings

Go to Cursor Settings → Tools & MCP → click "Add new MCP server"

2

Add SpecLock config

Add to your .cursor/mcp.json:

{ "mcpServers": { "speclock": { "command": "npx", "args": ["-y", "speclock", "serve", "--project", "."] } } }
3

Start Coding

In Cursor's AI chat, type: "Use SpecLock to initialize this project and set constraints"

Windsurf

Easy — 2 min
1

Open MCP Configuration

Go to Windsurf Settings → Cascade → MCP Servers

2

Add SpecLock Server

Add to your ~/.codeium/windsurf/mcp_config.json:

{ "mcpServers": { "speclock": { "command": "npx", "args": ["-y", "speclock", "serve", "--project", "."] } } }

Cline (VS Code Extension)

Easy — 2 min
1

Open Cline MCP Settings

Click the MCP Servers icon in the Cline sidebar → click "Configure MCP Servers"

2

Add SpecLock

Add to the MCP settings JSON:

{ "mcpServers": { "speclock": { "command": "npx", "args": ["-y", "speclock", "serve", "--project", "."] } } }

OpenAI Codex

Easy — 2 min
1

Via Smithery

Visit smithery.ai/servers/sgroy10/speclock → click Codex and follow the auto-generated configuration.

bolt.diy (Open Source Bolt)

Medium — 5 min
1

Open MCP Settings

In your bolt.diy instance, go to Settings → MCP

2

Add Remote MCP Server

Add SpecLock as a Streamable HTTP server:

{ "speclock": { "type": "streamable-http", "url": "https://speclock-mcp-production.up.railway.app/mcp" } }
