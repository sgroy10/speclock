SpecLock: The MCP Constraint Engine for AI-Safe Development

By Sandeep Roy · April 9, 2026 · 5 min read

The Model Context Protocol (MCP) has become the standard interface for AI coding tools to interact with external services. Claude Code, Cursor, Windsurf, Cline, and a growing number of editors support MCP servers that extend what the AI can do: access databases, query APIs, manage infrastructure.

But there's a category of MCP server that's been missing from the ecosystem: constraint servers. Tools that don't give the AI more capabilities, but rather define the boundaries of what it's allowed to do. SpecLock is the first dedicated MCP constraint engine, and this article is a technical deep-dive into how it works.

Architecture Overview

SpecLock runs as both a local MCP server (via stdio) and a remote HTTP server. The local server integrates directly with your AI coding tool. The HTTP server supports remote MCP connections and provides a health/status API.

# Local MCP server (stdio transport)
npx speclock --stdio

# Remote HTTP server
npx speclock serve --port 3001

# Or just protect your project (does everything)
npx speclock protect

When an AI tool connects to SpecLock via MCP, it gains access to 51 tools organized into functional categories. These tools operate on a constraint store that lives alongside your codebase in a .speclock/ directory.

The 51 Tools: What They Do

SpecLock's tools fall into six categories:

Constraint Management (Core)

speclock_add_lock Create a constraint from natural language. "Never use jQuery" becomes a typed, enforceable lock.
speclock_add_typed_lock Create a constraint with explicit type: TECHNOLOGY, ARCHITECTURE, PATTERN, SECURITY, etc.
speclock_remove_lock Remove a constraint (with audit trail). Requires explicit intent.
speclock_override_lock Temporarily override a constraint with a reason. Logged for compliance.

Conflict Detection

speclock_check_conflict Check if a proposed change conflicts with any active constraint. Returns confidence level.
speclock_review_patch Review a code patch against all constraints. Returns per-constraint results.
speclock_review_patch_diff Review a git diff directly. Integrates with pre-commit hooks.
speclock_blast_radius Estimate impact of a proposed change across the constraint graph.

Monitoring and Compliance

speclock_detect_drift Identify constraints that the codebase has drifted away from.
speclock_drift_score Quantitative drift metric (0-100). Higher means more constraint violations.
speclock_semantic_audit Full semantic analysis of codebase against all constraints.
speclock_export_compliance Export compliance report for stakeholders.

Rule Discovery

speclock_discover_rules Scan CLAUDE.md, .cursorrules, AGENTS.md and extract rules automatically.
speclock_sync_rules Sync constraints from rule files. Handles additions, removals, and modifications.
speclock_suggest_locks Analyze codebase and suggest constraints based on patterns detected.
speclock_compile_spec Compile all constraints into a unified spec document.

Session Management

speclock_session_briefing Load all project context at session start. Goal, constraints, decisions, history.
speclock_session_summary Summarize what happened in this session for future context.
speclock_set_goal Set or update the project's high-level goal.
speclock_add_decision Record an architectural or technology decision with rationale.

Analysis and Graphs

speclock_build_graph Build a constraint dependency graph showing relationships between locks.
speclock_coverage Measure what percentage of your codebase is covered by constraints.
speclock_map_locks Map constraints to files and directories they affect.
speclock_policy_evaluate Evaluate a change against policy rules (beyond individual constraints).

Semantic Detection: How It Actually Works

The core of SpecLock's value is semantic conflict detection. Here's what happens when check_conflict is called:

  1. Constraint parsing: The natural-language constraint is broken down into semantic components: subject (what it's about), verb (what action is constrained), object (what's affected), and polarity (must/must-not).
  2. Change analysis: The proposed change is analyzed for what it adds, removes, and modifies. This includes import analysis, function signature changes, dependency modifications, and structural changes.
  3. Semantic matching: The change components are matched against constraint components using semantic similarity rather than string matching. "Switching to Fastify" matches a constraint about "Always use Express" even though the words are different.
  4. Confidence scoring: Each match is scored as LOW, MEDIUM, or HIGH confidence based on the strength of the semantic connection and the specificity of the constraint.

This approach catches violations that string matching would miss entirely. A constraint like "Authentication must use JWT tokens" is violated by code that switches to session-based auth, even though the word "JWT" might not appear in the diff at all.

Integration Points

SpecLock integrates at three levels:

  • MCP (real-time): The AI tool checks constraints before generating code. This is the tightest integration, preventing violations before they happen.
  • Git hooks (pre-commit): Constraints are checked when code is committed. This catches violations that slipped through the real-time check.
  • CI/CD (pipeline): Compliance reports are generated as part of the build pipeline. This provides an audit trail and catches anything that bypassed local checks.

Why MCP Is the Right Protocol for Constraints

MCP was designed for tools that need to provide context and capabilities to AI models. Constraints are a natural fit because they are context that the AI needs to be aware of, and enforcement is a capability that the AI should delegate to a specialized server.

By implementing constraints as an MCP server, SpecLock gets automatic compatibility with every AI tool that supports MCP. No custom plugins for each editor. No API integrations to maintain. One server, universal compatibility.

Getting Started

# Install and protect in one command
npx speclock protect

# Or add to your MCP config manually
{
  "mcpServers": {
    "speclock": {
      "command": "npx",
      "args": ["-y", "speclock", "--stdio"]
    }
  }
}
